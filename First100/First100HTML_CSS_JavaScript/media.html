Media Query Syntax

Media queries use the @media rule to apply CSS only when certain conditions are met.

/* Basic Syntax */
@media screen and (max-width: 768px) {
  .container {
    padding: 15px;
  }
}

/* Multiple Conditions */
@media screen and (min-width: 768px) and (max-width: 1024px) {
  .container {
    max-width: 90%;
  }
}
Pro Tip: Always use a mobile-first approach. Start with styles for small screens, then use min-width queries to add styles for larger screens.

Media Query Types

Media queries can target different device characteristics and user preferences.

Screen Size

Target specific screen widths and heights

@media (max-width: 768px)
@media (min-height: 600px)

Color Scheme

Detect light or dark mode preference

@media (prefers-color-scheme: dark)
@media (prefers-color-scheme: light)

Media Types

Target specific output media

@media print
@media screen
@media speech

User Preferences

Respect user accessibility settings

@media (prefers-reduced-motion)
@media (prefers-contrast: high)

Device Orientation

Detect portrait or landscape mode

@media (orientation: portrait)
@media (orientation: landscape)

Resolution & Display

Target high-resolution displays

@media (-webkit-min-device-pixel-ratio: 2)
@media (resolution: 2dppx)

Common Breakpoints

Breakpoints are the screen sizes at which your design changes. Here are common breakpoints for responsive design.

/* Common Breakpoints */
/* Extra small devices (phones) */
@media (max-width: 576px) { }

/* Small devices (tablets) */
@media (min-width: 577px) and (max-width: 768px) { }

/* Medium devices (laptops) */
@media (min-width: 769px) and (max-width: 992px) { }

/* Large devices (desktops) */
@media (min-width: 993px) and (max-width: 1200px) { }

/* Extra large devices */
@media (min-width: 1201px) { }
Important: Don't design for specific devices! Design based on your content's needs. Use breakpoints where your content breaks, not based on popular device sizes.

Practical Examples

1. Mobile-First Navigation

/* Base styles (mobile first) */
.navbar {
  display: flex;
  flex-direction: column;
  padding: 1rem;
}

/* Tablet and larger */
@media (min-width: 768px) {
  .navbar {
    flex-direction: row;
    justify-content: space-between;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .navbar {
    padding: 2rem;
  }
}

2. Responsive Grid System

.grid {
  display: grid;
  gap: 1rem;
}

/* Mobile: 1 column */
@media (max-width: 767px) {
  .grid {
    grid-template-columns: 1fr;
  }
}

/* Tablet: 2 columns */
@media (min-width: 768px) and (max-width: 1023px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Desktop: 4 columns */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(4, 1fr);
  }
}

3. Dark Mode Support

/* Light mode (default) */
body {
  background-color: white;
  color: #333;
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  body {
    background-color: #1a1a1a;
    color: #f0f0f0;
  }
}

4. Print Styles

@media print {
  body {
    font-size: 12pt;
    line-height: 1.5;
  }
  .navbar, .footer, .ads {
    display: none;
  }
  a {
    color: black;
    text-decoration: none;
  }
  a::after {
    content: " (" attr(href) ")";
  }
}

Advanced Techniques

1. Container Queries (Modern CSS)

/* Define container */
.card-container {
  container-type: inline-size;
}

/* Style based on container size */
@container (min-width: 400px) {
  .card {
    display: flex;
    gap: 1rem;
  }
}
Container Queries vs Media Queries: Container queries allow elements to respond to their container's size, not just the viewport. This is perfect for reusable components.

2. Logical Operators

/* AND operator (default) */
@media screen and (min-width: 768px) and (max-width: 1024px) { }

/* OR operator (comma separated) */
@media (max-width: 600px), (min-width: 1200px) { }

/* NOT operator */
@media not screen and (color) { }

3. Responsive Typography with CSS Clamp

h1 {
  /* Minimum 2rem, ideal 5vw, maximum 3.5rem */
  font-size: clamp(2rem, 5vw, 3.5rem);
}

p {
  font-size: clamp(1rem, 2.5vw, 1.5rem);
}

Best Practices

1. Start with base styles for mobile
2. Use min-width queries for larger screens
3. Add enhancements progressively
4. Test at every breakpoint
