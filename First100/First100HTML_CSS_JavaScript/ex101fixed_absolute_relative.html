1. Relative Positioning

Key Point: Element is positioned relative to its normal position. The original space in the document flow is preserved.
position: relative;
top: 20px;
left: 30px;

2. Absolute Positioning

Key Point: Element is removed from normal document flow and positioned relative to its nearest positioned ancestor.
.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 40px;
  right: 20px;
}

3. Fixed Positioning

Key Point: Element is positioned relative to the viewport (browser window) and stays in place when scrolling.
position: fixed;
bottom: 30px;
right: 30px;
Fixed Positioning Demo

The purple "Fixed Element" box in the bottom-right corner of your screen is positioned with position: fixed.

Scroll this page and notice how the fixed element stays in the same position relative to your browser window.

Fixed elements are commonly used for:

  • Navigation bars
  • Chat widgets
  • Back-to-top buttons
  • Cookie notices

Comparison of Positioning Types

Position Type Relative To In Document Flow Common Uses
Static Normal flow position Yes Default for all elements
Relative Its normal position Yes (space preserved) Minor adjustments, positioning context for absolute children
Absolute Nearest positioned ancestor No Overlays, tooltips, custom controls
Fixed Viewport (browser window) No Sticky headers, floating buttons, modals

Practical Example: Combining Positioning Types

/* Common pattern: parent relative, child absolute */
.tooltip-container {
  position: relative;
  display: inline-block;
}

.tooltip {
  position: absolute;
  bottom: 100%;
  left: 50%;
  transform: translateX(-50%);
}
Remember: For absolute positioning to work relative to a parent, the parent must have a position value other than static (usually relative).
