@keyframes Syntax

The @keyframes rule defines the animation sequence by specifying styles at various points during the animation.

/* Basic Syntax */
@keyframes animation-name {
  /* From/To syntax */
  from { property: value; }
  to { property: value; }
}

/* Percentage syntax */
@keyframes animation-name {
  0% { property: value; }
  50% { property: value; }
  100% { property: value; }
}
Tip: You can use from and to as aliases for 0% and 100%. For complex animations, use percentage values to define intermediate steps.

Animation Properties

These properties control how the animation behaves. They can be set individually or using the animation shorthand.

animation-name

Specifies the name of the @keyframes animation.

animation-name: slideIn;
animation-duration

How long one animation cycle takes.

animation-duration: 2s;
animation-timing-function

How the animation progresses through each cycle.

animation-timing-function: ease-in-out;
animation-delay

Time to wait before starting the animation.

animation-delay: 1s;
animation-iteration-count

How many times the animation runs.

animation-iteration-count: infinite;
animation-direction

Whether animation runs normally, reversed, or alternates.

animation-direction: alternate;
animation-fill-mode

What styles apply before/after animation.

animation-fill-mode: forwards;
animation-play-state

Whether animation is running or paused.

animation-play-state: paused;
/* Shorthand syntax (order matters!) */
animation: name duration timing-function delay iteration-count direction fill-mode play-state;

/* Example */
animation: slideIn 2s ease-in-out 0.5s infinite alternate forwards;

Common Animation Examples

These are some of the most commonly used animations in web design.

1. Fade In

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

2. Slide In

@keyframes slideInLeft {
  from { transform: translateX(-100px); }
  to { transform: translateX(0); }
}

3. Bounce

@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-30px); }
}

4. Rotate

@keyframes rotate {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

5. Pulse

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.1); }
}

6. Shake

@keyframes shake {
  0%, 100% { transform: translateX(0); }
  10%, 30%, 50%, 70%, 90% { transform: translateX(-10px); }
  20%, 40%, 60%, 80% { transform: translateX(10px); }
}

Performance & Best Practices

Good for Performance Bad for Performance transform: translate, scale, rotate width, height, margin, padding opacity box-shadow (heavy use) will-change: transform (sparingly) Too many simultaneous animations Using requestAnimationFrame for JS animations Using setInterval for animations GPU accelerated properties Properties that cause layout reflow
Performance Tip: Always animate with transform and opacity when possible. These properties can be hardware-accelerated by the GPU, making animations smoother and more efficient.
A11Y Warning: Respect user preferences with @media (prefers-reduced-motion: reduce). Some users may experience motion sickness or distractions from animations.

Advanced Techniques

1. Multiple Animations

.element {
  /* Apply multiple animations */
  animation:
    slideIn 1s ease,
    fadeIn 2s ease,
    colorChange 3s linear infinite;
}

2. Step Animations

@keyframes stepsAnimation {
  0% { background-position: 0 0; }
  100% { background-position: -1600px 0; }
}

.sprite {
  animation: stepsAnimation 1s steps(8) infinite;
}

3. Animation Events (JavaScript)

// JavaScript animation events
element.addEventListener('animationstart', () => {
  console.log('Animation started');
});

element.addEventListener('animationiteration', () => {
  console.log('Animation repeated');
});

element.addEventListener('animationend', () => {
  console.log('Animation ended');
});
